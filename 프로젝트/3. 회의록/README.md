# bigdatalearning - 회의록

### 2019.07.26
```
회의 내용
1. 다음주부터 딥러닝, 자연어 처리과정이 들어가지만, 데이터 수집을 진행해야함.
-> 주말동안 맘카페 데이터를 일부분 크롤링하여 실습이나 복습 자료로 써볼 수 있음.
-> 크롤링 봇은 자동화 시키는 대신에 만약 데이터 수집하는 형태(예.. 정규식)가 변할 것을 대비하여 일 부분을 쉽게 수정할 수 있게 구현해야함.

2. 분석을 나중에 진행한다해도 현재 단계에서 서비스에 대한 구체화 계획이 필요할 거 같음.
-> 주말을 이용하여 실제 서비스를 하는 페이지라고 생각하고 전체 흐름도 각자 그려옴. (웹 프로토타입이나, 직직접 필기로)

3. 백엔드 프론트엔드에 대해 정확한 역할 배분 필요
-> 이 부분에 대해서는 질문을 하려고 함.


보완 사항
1. 시스템 즉 서비스 구체화
2. 발표 자료 디자인에 투자
3. 단어에 대한 감정 분석을 할 시 정확한 오차 범위를 줄여 자연어 처리를 해야함. (즉 기술적인 부분에서도 구체화 시켜야함.)
4. 교육 과정치고는 분석이 차지하는 비용이 적음. (분석할 수 있는 요소를 생각도 해보고, 서비스 부분에 좀 더 연구해야할 필요가 있을 거 같음.)

```

```
회의 내용
1. 웹 서비스 화면 구상
-> 정리해서 올릴 예정

2. 서비스 요소 생각
-> 지역활성화에 초점을 둬야한다.
-> 세부적인 지도를 활용하여 해당 위치(예를 들면 가게)에 대한 리뷰나 글들을 정리해 줌.
-> 만약 회원가입, 로그인을 만든다면 그만큼 시간 소모가 큼, 하지만 여러가지 서비스를 넣을 수 있음. 예를 들면 원하는 시간대의 분석 결과를 메일로 보내줌.(이는 어떤 형식으로 써야 할지 고민해 봐야함.)

3. 데이터 수집
-> 네이버 카페 같은 경우 iframe이 웹페이지를 클릭해야지 주소가 나타나는 시스템이라서 모바일 홈페이지로 시도함. => 그 결과 데이터 수집 할 수 있음.

4. 다른 주제에 대한 생각
-> 사람들이 치과도 전문의에게 치료 받고 싶어하는 경향이 있음. 그렇기에 4000개 정도 되는 치과 홈페이지를 크롤링하여 의사에 대한 정보를 모은 후 그 의사가 쓴 논문을 자연어 처리를 통해 나온 결과를 활용하여 사용자랑 매치 시켜주는 시스템 개발.

###### 일단 다른 주제같은 경우 검토맡고 진행할 예정
```
